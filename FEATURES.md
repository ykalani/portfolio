# Vibe OS: Retro Windows Portfolio MVP Features

This document provides a comprehensive scope and explanation of the fully functional retro Windows desktop portfolio (Vibe OS).

## 1. Desktop Shell & Window Manager

The outer presentation layer functions like a 16-bit operating system shell in the browser.

*   **Window Management:** Draggable, focusable, minimizable, and closable window frames. Tapping a window raises its `zIndex` to the top.
*   **Taskbar integration:** Reflects running apps with button triggers. Allows focusing, minimizing, or restoring windows.
*   **Start Menu:** Quick access links to portfolio sections (About, Projects, Experience, Contact, Resume) and desktop controls (Reset Desktop, Minimize All).
*   **Aesthetics:** Modern, tactile retro borders, HSL gradient title bars, and 3D bevel buttons. Built with custom SVG assets (`button_rectangle_depth_gradient.svg`, `input_outline_rectangle.svg`, `divider.svg`, `icon_play_light.svg`, `star.svg`, `icon_arrow_up_light.svg`).
*   **State Persistence:** Active layout positioning, open windows list, and generated windows list are automatically serialized and saved to `localStorage` on change.

---

## 2. App Forge Launcher

The desktop entry point allows users to generate fully active, responsive applications from text descriptions.

*   **Natural Language Mapping:** Accepts user prompts and parses keywords to determine the application "kind".
*   **Manifest Contract:** Maps queries to JSON-schema manifests containing title, kind, icon glyph, size dimensions, and custom color accents.
*   **Onboarding examples:** A row of click-to-run prompt chips (e.g., "build a retro calculator", "launch a task tracker").

---

## 3. Fully Operational Application Presets

Every app kind generated by the launcher is interactive and fully functional, rather than just displaying static mock panels.

### 📊 Command Center (Dashboard)
An operations dashboard showing live diagnostic feedback.
*   **Interactive Controls:** Slider to adjust simulated request traffic; Load Profile dropdown selector (Idle, Normal, Stress Test); Toggles to enable Firewall Security and CDN Edge caching.
*   **Live Charts:** Visual bar chart constructed using pure CSS elements, with height bars that transition dynamically according to load rates.
*   **System Event Logs:** A terminal-like scrollable log showing simulated edge transactions. Enabling firewall or CDN prints custom log timestamps in real time.

### ⏱ Focus Timer (Pomodoro)
A time-boxing utility for work cycles.
*   **Timer Controls:** Play, Pause, and Reset controls.
*   **Duration Presets:** Quick buttons for Pomodoro (25 mins), Short Break (5 mins), and Long Break (15 mins).
*   **Sound Synthesis:** Synthesizes an alarm tone on completion using the browser's HTML5 **Web Audio API** (creating an audio context and oscillator on the fly) to guarantee sound plays without external audio file dependencies.
*   **Visual Indicators:** CSS progress bar displaying ratio of time elapsed.

### 💬 Chat Client (Portfolio Assistant)
An assistant chatbot that helps users learn about the portfolio creator.
*   **Dynamic Knowledge Base:** Resolves questions regarding skills, experience, projects, resume details, and contact info, fetching values directly from `content.js` to ensure the bot is always in sync with portfolio edits.
*   **Typing Simulator:** Mimics compiling latencies with animated dots to give a natural conversational feel.

### 🌐 Web Viewer (Browser)
A web viewing shell that simulates surfing pages.
*   **Chrome Header:** Back, Forward, Refresh navigation buttons and URL Address bar supporting Enter key actions.
*   **Simulated Sites:** Avoids iframe blocks (CORS) by rendering custom mock pages for:
    *   `google.com`: Google search bar that yields search results matching the owner's portfolio sections.
    *   `github.com`: Displays repositories, descriptions, language badges, and follower tallies.
    *   `linkedin.com`: Profile card with connection prompts, professional experience timeline, and bio.
    *   `vibe://welcome`: Standard operating system release notes.
*   **External Sandbox:** Safely falls back to a sandbox iframe for actual external URLs with warning dialogs.

### 📝 Notepad (Notes)
A split-pane editor to manage documents.
*   **Sidebar List:** Shows all saved note titles. Includes a `+ New Note` button.
*   **Editing Workspace:** Input fields for Note Title and note textarea content.
*   **Saving/Deleting:** Saves content to local memory, updates note list titles instantly, and deletes notes from the stack, persisting updates to `localStorage`.

### ☑ Task Board (Todo)
A task-tracking checklist.
*   **Task List:** Checkboxes to strike through and mark tasks complete, and delete buttons.
*   **Input Box:** Adds new checklist items to state on form submit.
*   **Filters:** Quick selectors to toggle list view between "All", "Active", and "Completed" tasks.
*   **Clear Action:** Clear all completed tasks at once.

### 🧮 Calculator
A 16-bit utility for calculations.
*   **LCD Display:** Green calculator output window.
*   **Arithmetic Keys:** Grid of buttons for numbers, parenthesis, clear, backspace, operators (`/`, `*`, `-`, `+`), and evaluation.
*   **Evaluation Engine:** Safe formula parsing and evaluation preventing arbitrary code execution.

## 4. Multi-Threaded Simulator Engine

Vibe OS runs a background process loop matching the ticking timer frequency (every 1000ms):

1.  **Clock Sync:** Syncs the taskbar digital clock.
2.  **Focus Timer Tick:** Decrements active Pomodoro timers, triggers alerts, and pushes alarm records to the system logger.
3.  **Dashboard Load Fluctuation:** Simulates dynamic CPU load, memory usage, and edge network transactions in the background, updating the dashboard DOM in real time if open.
4.  **Targeted DOM Updates:** Modifies only the interior DOM nodes of open windows (instead of rerendering the whole desktop shell), ensuring inputs do not lose focus or text selections.
